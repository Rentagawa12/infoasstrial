🎯 Project Overview

Lost & Found @ TIP Manila is a comprehensive system designed to help TIP Manila students report and locate lost items within the campus community. The system integrates multiple modules including authentication, item management, notifications, analytics, and API key management.

Item Management

  • Item name and description
  • Category (Electronics, Accessories, Documents, Clothing, Books, Others)

🔗 System Integration

Module Interactions

The system uses an event-driven architecture with message queue simulation for cross-module communication:

ITEM CREATED EVENT
└─> Item Module
    ├─> Notification Module (Check for matches, send alerts)
    ├─> Analytics Module (Log item_posted event)
    └─> Orchestration Layer (Broadcast to all handlers)

ITEM CLAIMED EVENT
└─> Item Module
    ├─> Notification Module (Notify item owner)
    ├─> Analytics Module (Log item_claimed event)
    └─> Orchestration Layer

USER REGISTERED EVENT
└─> Auth Module
    ├─> Notification Module (Send welcome message)
    ├─> Analytics Module (Log user_registered event)
    └─> Orchestration Layer

Event Bus System

The orchestration middleware manages inter-module communication:

  • item:created - Emitted when new item is posted
  • item:claimed - Emitted when item is marked as claimed
  • item:expiring - Emitted 24hrs before item expiration
  • user:registered - Emitted on user signup
  • user:login - Emitted on user login
  • security:alert - Emitted on security events

Database Relationships

Collections are related via ObjectId foreign keys with application-level integrity:

  • Users → Items (1:N) - One user can post multiple items
  • Users → Notifications (1:N) - One user receives many notifications
  • Items → Notifications (1:N) - One item can trigger multiple notifications
  • Users → Analytics Events (1:N) - One user generates many events
  • Users → API Keys (1:N) - One user can have multiple API keys

🔒 Security Implementation

Authentication (JWT)

  • JWT tokens expire after 24 hours
  • Passwords hashed with bcryptjs (salt rounds: 10)
  • Token stored in Authorization header: Bearer {token}
  • JWT payload includes userId and role

Authorization (RBAC)

Role-Based Access Control with three roles:

Role Can Read Can Write Can Delete Can Admin
User ✅ Own items ✅ Own items ❌ ❌
Moderator ✅ All items ✅ All items ✅ Review ❌
Admin ✅ All ✅ All ✅ All ✅ Yes

Rate Limiting

Endpoint Type Limit Window
Public Endpoints 100 requests 15 minutes
Auth Endpoints 10 attempts 15 minutes
Authenticated Endpoints 500 requests 15 minutes

Input Validation & Sanitization

  • XSS Protection: HTML entity encoding, CSP headers
  • MongoDB Injection: Schema validation, operator sanitization
  • SQL Injection: N/A (MongoDB used)
  • CSRF: CORS configuration, same-site cookies
  • File Upload: MIME type validation, size limits (5MB)

Security Headers

Content-Security-Policy: default-src 'self'
X-Frame-Options: DENY
X-Content-Type-Options: nosniff
X-XSS-Protection: 1; mode=block
Strict-Transport-Security: max-age=31536000
Referrer-Policy: no-referrer

Data Protection

  • Passwords never logged or transmitted in plain text
  • Sensitive data excluded from responses (passwords, API keys)
  • TTL indexes for automatic cleanup of sensitive data
  • API keys hashed with SHA256

🚀 Deployment & DevOps

CI/CD Pipeline (GitHub Actions)

Automated testing and deployment on push to main branch:

  1. Test Stage: Run Jest integration tests with MongoDB
  2. Build Stage: Build Docker image and push to Docker Hub
  3. Deploy Stage: Trigger Render.com deployment webhook

Docker Configuration

# Multi-stage build
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm install

FROM node:18-alpine
WORKDIR /app
COPY --from=builder /app/node_modules ./node_modules
COPY . .
EXPOSE 10000
CMD ["node", "BACKEND/server.js"]

❓ Frequently Asked Questions

How do I report a lost item?

1. Click "Get Started" on the home page
2. Enter your student ID and name
3. Fill in the item details, select category and status
4. Upload a photo (optional)
5. Click "Submit Item"

How long are items stored in the system?

Items are automatically removed after 30 days. After this period, you can repost the item if needed.

Can I edit my item report after posting?

Currently, you cannot edit items directly. Please contact an admin to modify or delete an incorrectly posted item.

How do I get notified about matching items?

When you post a found item, the system automatically notifies anyone who posted a similar lost item. Check the notification section for updates.

What image formats are supported?

Supported formats: JPG, JPEG, PNG, GIF, WebP. Maximum file size: 5 MB.

Is my contact information private?

Yes, your name and contact information are only visible to other users who are also browsing the lost & found items. They will not be shared with anyone else.

What are API keys used for?

API keys allow third-party applications to access the Lost & Found system data. They can be generated in the admin panel with specific permissions.

How do I report a security issue?

Please email admin@lostandfound.tip.edu.ph with details. Do not post security vulnerabilities publicly.

Can I access the system via mobile?

Yes! The system is fully responsive and works on all devices including phones and tablets.

What happens if I mark an item as "Claimed"?

Marking an item as claimed removes it from the active lost & found listings and archives it for record-keeping purposes.
